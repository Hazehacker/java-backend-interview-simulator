# 评分标准与反馈模板

## 评分维度

| 维度 | 权重 | 等级说明 |
|------|------|----------|
| 技术深度 | 30% | 对技术原理的理解程度，能否触达底层 |
| 项目经验 | 30% | 项目复杂度、解决实际问题的能力、工程思维 |
| 思维逻辑 | 20% | 问题分析和解决思路的清晰度、条理性 |
| 表达能力 | 10% | 技术表达的准确性、逻辑性、流畅度 |
| 学习能力 | 10% | 对新技术的理解和应用能力、AI 工具使用经验 |

---

## 评分等级参考

### 技术深度（30%）

| 等级 | 分数段 | 表现 |
|------|--------|------|
| 优秀 | 9-10 | 能讲清底层原理（JVM 源码、MySQL 索引结构、CAS 机制等），有系统性理解 |
| 良好 | 7-8 | 对核心原理理解正确，能回答"为什么" |
| 一般 | 5-6 | 知道基本概念，但停留在使用层面 |
| 不足 | 3-4 | 概念模糊，无法准确描述 |
| 较差 | 1-2 | 完全不了解，多次提示后仍无法回答 |

### 项目经验（30%）

| 等级 | 分数段 | 表现 |
|------|--------|------|
| 优秀 | 9-10 | 项目规模大（千万级用户/高并发/复杂业务），能说清架构选型理由和权衡 |
| 良好 | 7-8 | 有一定规模的项目，能描述技术难点和解决方案 |
| 一般 | 5-6 | 项目较小，但能说明自己的贡献和收获 |
| 不足 | 3-4 | 项目描述模糊，无法回答追问 |
| 较差 | 1-2 | 无真实项目经验或完全虚构 |

### 思维逻辑（20%）

| 等级 | 分数段 | 表现 |
|------|--------|------|
| 优秀 | 9-10 | 思路清晰，能层层拆解问题，给出完整的分析路径 |
| 良好 | 7-8 | 逻辑连贯，能有条理地表达观点 |
| 一般 | 5-6 | 基本能回答，但思路有时跳跃 |
| 不足 | 3-4 | 思维混乱，前后矛盾 |
| 较差 | 1-2 | 无法形成有效回答 |

### 表达能力（10%）

| 等级 | 分数段 | 表现 |
|------|--------|------|
| 优秀 | 9-10 | 表达流畅、专业术语使用准确，能用类比辅助解释 |
| 良好 | 7-8 | 表达清晰，术语基本准确 |
| 一般 | 5-6 | 能表达基本意思，但有时词不达意 |
| 不足 | 3-4 | 表达困难，需要多次追问才能理解 |
| 较差 | 1-2 | 无法有效沟通 |

### 学习能力（10%）

| 等级 | 分数段 | 表现 |
|------|--------|------|
| 优秀 | 9-10 | 对 AI 工具（GitHub Copilot、Cursor 等）有深度使用和思考，关注技术趋势 |
| 良好 | 7-8 | 有实际使用经验，能说明如何将 AI 融入工作流 |
| 一般 | 5-6 | 了解 AI 工具但使用较少 |
| 不足 | 3-4 | 对 AI 工具几乎不了解 |
| 较差 | 1-2 | 对新技术完全无感 |

---

## 综合评分计算

```
综合评分 = 技术深度×0.3 + 项目经验×0.3 + 思维逻辑×0.2 + 表达能力×0.1 + 学习能力×0.1
```

**最终分数保留 1 位小数**

---

## 反馈模板

面试结束后，按以下模板输出反馈：

```
──────────────────────────────────────
         面试评估报告
──────────────────────────────────────

【综合评分】X.X / 10

【分项评分】
├─ 技术深度    X.X / 10（权重 30%）
├─ 项目经验    X.X / 10（权重 30%）
├─ 思维逻辑    X.X / 10（权重 20%）
├─ 表达能力    X.X / 10（权重 10%）
└─ 学习能力    X.X / 10（权重 10%）

──────────────────────────────────────

【优点】
  ✓ [具体优点 1，引用候选人的回答作为佐证]
  ✓ [具体优点 2]

【待改进】
  △ [具体改进点 1]
  △ [具体改进点 2]

【下一步学习建议】
  → [针对性推荐学习内容或方向]

【面试官留言】
  [2-3 句鼓励或建设性的话]

──────────────────────────────────────
```

## 面试进行中提示（不透露分数，只给引导）

| 场景 | 面试官话术 |
|------|-----------|
| 回答模糊时 | "能具体展开一下吗？" / "这个点你是怎么考虑的？" |
| 遇到困难时 | "没关系，我们可以换个角度想想……" / "提示一下，这个问题涉及 XX 原理" |
| 回答较好时 | "很好，继续" / "理解正确，能再深入一点吗？" |
| 压力测试时 | "确定吗？" / "还有其他可能的方案吗？" |
| 时间紧张时 | "由于时间关系，这个问题先到这里" |
